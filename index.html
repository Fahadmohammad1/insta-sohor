Question About Javascript

Javscript runs in a engine which is called V8 engine. javascript source code goes to the engine. V8 engine parse the code and make Abstract Syntax Tree (AST). After that the code will compiled to the Byte code. than byte codes are compiled to machine code by Just in Time compiler (JIT). The last step is machine code runs on the CPU. That's how javascript work.

Event loop help javascript to do multiple work. In V8 engine there are two things called heap and call stack. heap is a region of memory. when we call a function it will go to the call stack. Which is called stack of frame. than it will execute called function one by one. But it execute the last called function first. that's why it's called "last in first out (LIFO)". When we call a asynchronous function like (setTimeout , setInterval, fetch) call stack send it to Web Api and continue other function.Web API send the function to queue after complete the work. After completing all frame event loop check if the call stack free or not. If call stack is free its send the queue funtion to call stack. When the task of queue is completed it will send a signal to the call stack. And event loop carry that signal. After completing the all frame call stack started to execute the queue function. Queue work as "first in first out (FIFO)". that means first function enterd in the queue will send to call stack first and executed first.

Local Storage:

  • localStorage stores data permanently. It will not expire even we close the browser. We can get our data back even after more than a year.
  • localStorage data can be deleted only by clearing the cache memory of browser and also by javascript. It is a client side storage.
  • Memory of localStorage is 10 MB. But some people says it is only 5 MB.

Session Storage:

  • sessionStorage data is temporary. It wiil be vanished after we close the browser.
  • session storage data work for a session only. When user close the tab or window it will delete the data automatically. It is also a client side storage.
  • sessionStorage size is 5MB
